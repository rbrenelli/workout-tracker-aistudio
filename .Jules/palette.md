## 2024-06-27 - [A11y Pattern] Custom Accordion Divs Lacking Keyboard Support
**Learning:** Found a common anti-pattern in the codebase where `div` elements act as interactive accordion headers (e.g., `ExerciseCard` and `WorkoutHistoryView`) but entirely lack `role="button"`, `tabIndex={0}`, keyboard event handlers (`onKeyDown`), and visual focus outlines (`focus-visible`). This makes the application completely inaccessible for keyboard-only users who cannot expand details or history logs. Additionally, input fields and buttons were missing clear `aria-label`s and `focus-visible` styling.
**Action:** When working with expanding/collapsing custom components or complex interactive cards in this React/Tailwind codebase, always verify if it's purely a `div` with an `onClick`. If so, mandate the addition of proper semantic roles, `tabIndex`, keydown handlers (Enter/Space), and visible focus rings using Tailwind's `focus-visible:` utilities.
