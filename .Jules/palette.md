## 2024-10-18 - Missing label association and ARIA states on custom checkbox inputs
**Learning:** Custom interactive elements (like weight input fields and checkmarks acting as checkboxes) inside repetitive elements like exercise lists must have linked text labels via `id`/`htmlFor` and explicit `aria-pressed` / dynamic `aria-label` states. Because they act as the primary interaction on mobile and desktop, leaving them untagged impacts screen-reader accessibility for the core functionality. Also, visually decorative suffixes (like "kg") must be hidden from screen readers.
**Action:** Always link a descriptive `<label>` visually adjacent to an `<input>` for input contexts. Always use `aria-pressed` (with boolean value) for custom button toggles mimicking checkboxes, and dynamically adjust `aria-label` to provide context (e.g. "Check" vs "Uncheck"). Ensure a `focus-visible` ring is applied for keyboard navigation, particularly in complex UI cards.
