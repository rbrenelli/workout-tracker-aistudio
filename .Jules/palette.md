## 2024-05-24 - Form Control Links and Decorative Text Hiding
**Learning:** In complex form-like components where labels appear disjointed visually from their inputs (e.g. nested inside spans), relying on the input context alone harms screen reader accessibility. Additionally, inline unit metrics (like "kg") or static text that changes based on state (like "CONCLUÍDO") can add unnecessary noise when read alongside interactive element states, unless correctly grouped or hidden.
**Action:** When building interactive UI elements with nested inputs, always explicitly link text labels to inputs using `<label htmlFor="...">` and `id`, rather than relying on standard DOM order or wrappers. Moreover, aggressively use `aria-hidden="true"` on unit indicators ("kg") and decorative text spans that duplicate toggle-state information to ensure a clean, focused readout.
